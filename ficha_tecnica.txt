ECMatic Ficha Técnica de Producto — CRM con IA para Centro de Evaluación CONOCER Versión 1.0 · Junio 2026 · Confidencial 1. Visión General del Producto ECMatic es un CRM propietario diseñado exclusivamente para Centro ECM (Ojos Abiertos MX), que centraliza la adquisición, calificación, cierre y retención de candidatos a certificación CONOCER. Su diferenciador central es una base de conocimiento viva y dinámica, gobernada por IA, que se alimenta de conversaciones reales con leads y actúa como cerebro operativo de todas las comunicaciones del negocio. Dominio de producción: ceecm.mx Stack: Next.js (App Router) · Tailwind CSS · shadcn/ui · Supabase · pgvector · WhatsApp Business API (Meta) · Resend (transaccional) · Brevo (nurturing) · Stripe · Google Calendar · Google Meet · SmartBuilderEC API Principio de código: ningún archivo supera 300 líneas. Extracción agresiva de componentes. Cero lógica de negocio en capas de UI. 2. Principios de Desarrollo con Claude Code Máximo 300 líneas por archivo. Si un archivo crece más, extraer componentes o utilidades antes de continuar. Cero lógica de negocio en componentes de UI. Los componentes reciben datos y disparan callbacks. La lógica vive en hooks, servicios o API routes. Cada subsprint es un commit atómico verificable. No avanzar al siguiente subsprint sin que el entregable del anterior esté funcionando. Todas las llamadas a APIs externas tienen manejo de errores, reintentos con backoff y logging estructurado. Las variables de entorno nunca se hardcodean. Usar siempre process.env con validación al inicio de la aplicación. Los embeddings se regeneran automáticamente al crear o actualizar cualquier recurso de la base de conocimiento. La IA nunca ejecuta acciones irreversibles sin aprobación del admin (borrar recursos, reasignar leads, cambiar precios). Los roles de Supabase (RLS) se configuran en el mismo sprint donde se crea la tabla. Nunca dejar tablas sin políticas de seguridad. El dominio ceecm.mx apunta a producción. Las pruebas de integraciones críticas (WhatsApp, Stripe) se hacen en entornos de sandbox/test antes de activar en producción. La arquitectura es agnóstica de proveedor de IA. El router de selección de modelo es un módulo independiente configurable desde el panel. ECMatic — Documento generado a partir de entrevista de 100 preguntas.
